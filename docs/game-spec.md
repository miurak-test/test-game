## 3. ゲームシステム

### 3.1 幸福度

| 表示軸 | 内部の柱 | 説明 |
|--------|---------|------|
| 🌿 くらし | 自然・健康 + お金・安定 | 生活の土台 |
| 💬 つながり | 社交・友情 | 人との関係 |
| ✨ じぶん | 創造・達成 + 趣味・文化 | 自己実現 |

| メカニクス | 効果 |
|-----------|------|
| **疲労** | 良いイベント → fatigue +1。fatigue ≥ 3 で正の変動を `max(0, raw - floor(fatigue/3))` に減衰 |
| **洞察** | 悪いイベント → insight +1。insight ≥ 5 で creation/culture に `floor(insight/5)` ボーナス |
| **バランスボーナス** | 3 軸すべて ≥ 8 で `floor(min / 2)` の追加ポイント |
| **休息** | 💤 マスで疲労 -3 |
| **燃え尽き** | fatigue ≥ 100 で自動発生。fatigue を 50 にリセット + 洞察 +10 |

### 3.2 ターンの流れ（4 拍子固定）

```
🎲 サイコロ → 🚶 移動 → 📖 イベント/選択 → 📊 決算
```

### 3.3 運命のダイス（倍プッシュ）

| 項目 | 内容 |
|------|------|
| 発動条件 | 分岐 / 選択マスで洞察 ≥ 5 のとき「運命に賭ける」ボタン出現 |
| 成功（2d6 ≥ 7、58%） | 全効果 3 倍 + 称号「運命の賭博師」 |
| 失敗（2d6 ≤ 6、42%） | 疲労 +30 + 最も好感度の高い NPC -10 + 洞察 +3 |
| 制限 | 季節に 1 回のみ |
| 演出 | BGM が「異常」に切替。成功 → 派手エフェクト、失敗 → 暗転 |

**設計思想：** 「悪いイベントを引いた者だけが、一発逆転の権利を得る」。賭けた経験自体に価値がある（失敗時も洞察 +3）。

### 3.4 ボード構造（蛇行レイアウト）

```
春（左→右）→ 夏（右→左）→ 秋（左→右）→ 冬（右→左）→ GOAL
```

各季節：メイン 2 マス → 分岐 → ルート 2 マス → 合流 / 祭り

| マス | アイコン | 内容 |
|------|---------|------|
| 生活 | 🌱 | 日常イベント |
| 選択 | ❓ | 2 択のトレードオフ |
| 分岐 | 🔀 | 農業（疲労軽減）/ お店（お金+1）/ 村（洞察+2） |
| 祭り | 🎉 | 年鑑に記録 |
| 休息 | 💤 | 疲労 -3 |
| ハプニング | ⚡ | 洞察 UP のチャンス |
| 恋愛 | 💕 | NPC との交流・修羅場の誘惑 |

| モード | マス数 | 周回 | プレイ時間 |
|--------|-------|------|-----------|
| ショート | 約 20 マス | 1 周（1 年） | 10-15 分 |
| フル | 約 60 マス | 3 周（3 年） | 30-45 分 |

## 7. 人生診断カード + SNS シェア

### 診断カードの要素

| 要素 | 内容 |
|------|------|
| 人生グラフ | 幸福度の推移を曲線で表示 |
| キャッチコピー自動生成 | 最高点 →「奇跡の冬」、最低点 →「魔の夏休み」 |
| 人生カテゴリ判定 | 「自然派スローライフ」「波乱万丈ドラマ人生」「修羅場コレクター」等 |
| ハッシュタグ | `#恋ダイス #人生診断 #幸福度XX点` |

### 出力形式

| 形式 | 内容 |
|------|------|
| PNG | 静止画（人生グラフ + キャッチコピー） |
| GIF | 3-5 秒アニメ（グラフ描画 → キャッチコピーがドン！） |
| テキスト | Wordle 式（`⬛️🟩🟨 幸福度:85点 #恋ダイス`） |

### SNS シェア 3 段フォールバック

1. `navigator.share({files})` → 画像 / GIF 直接シェア
2. 失敗 → PNG / GIF DL ボタン
3. In-App ブラウザ → Wordle 式テキストコピー

## 9. AI アセット生成戦略（v2.0）

**原則：** ゲーム中はサクサク（事前生成）、リザルトだけ特別（動的 AI 生成）。

| タイミング | 生成方法 | 対象 |
|-----------|---------|------|
| 開発時 | Nano Banana 2 | NPC 立ち絵（感情差分 4 種 × 4 人 = 16 枚）、イベントスチル |
| 開発時 | Lyria 3 | BGM ステム（穏やかベース + 不穏ノイズ）→ Phaser で動的クロスフェード |
| リアルタイム | Gemini 2.5 Flash テキスト | 年鑑ナレーション + キャッチコピー（リザルト画面のみ） |
| リアルタイム | Gemini 画像生成 | 「世界に 1 枚の年鑑アート」（リザルト画面のみ） |

画風：90 年代レトロアニメ風。カラーパレット Hex 指定 + Image-to-Image で一貫性確保。

### Gemini 年鑑ナレーション

```
プロンプト：「あなたは皮肉屋の吟遊詩人です。プレイヤーの1年を140字以内で
ドラマチックかつ少し笑えるように要約してください。
修羅場や倍プッシュ失敗があれば最大限にイジってください。
【厳守】入力データ以外は絶対に参照しないこと。」

model: gemini-2.5-flash / temperature: 0.4 / maxOutputTokens: 200
ハルシネーション対策：出力検証 → 不正ならフォールバックテンプレート使用
コスト：~$0.10/月（1,000 DAU）
```

## 11. インフラ（GCP + Terraform）

### GCP 構成（v2.0 で導入）

| サービス | 用途 | コスト（1,000 DAU） |
|---------|------|-------------------|
| Firebase Hosting | CDN 配信 | 無料枠内 |
| Firebase Auth | 匿名認証 → ソーシャルリンク | 無料 |
| Firestore | レガシーシステムのクロスデバイス同期 | 無料枠内 |
| Gemini API | 年鑑ナレーション + 画像生成 | ~$0.10/月 |
| Firebase Analytics + BigQuery | ゲーム分析 | 無料 |
| Secret Manager | API キー管理 | 無料枠内 |
| Cloud Functions | Gemini API 連携 | 無料枠内 |

### Terraform 構成

```
infra/
├── environments/
│   ├── dev/    # miurak-dev（独立 Firebase プロジェクト）
│   └── prod/   # miurak（既存プロジェクト）
└── modules/
    ├── firebase-hosting/
    ├── firestore/
    ├── firebase-auth/
    ├── cloud-functions/
    ├── secret-manager/
    └── analytics/
```

- `terraform plan` / `terraform apply` は Atlantis 経由
- state は GCS バックエンド + state locking
- Firestore Security Rules は Firebase CLI（CI 経由）でデプロイ
- リソース命名：同種 1 つなら `this`

### セキュリティ

- Firestore：`hasOnly()` フィールドホワイトリスト + `serverTimestamp()` + Cloud Functions 経由書込み
- 書込み頻度制限：最終書込み時刻の差分チェック
- Gemini API：1 ユーザー 1 日 3 回まで（Firestore キャッシュ）
- コスト爆発防止：Budget Alert + Cloud Functions 最大インスタンス数制限

### 認証フロー

```
初回 → ログインなしでプレイ（localStorage）
  → リザルトで「データを保存しませんか？」
  → 匿名認証 → ソーシャルリンク
  → Firestore に書込成功 → localStorage にマイグレ完了フラグ
  （localStorage は削除しない。Firestore が master）
```
